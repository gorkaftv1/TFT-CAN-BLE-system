{\huge{\textbf{Resumen}}}

    {\large{El presente Trabajo de Fin de Título (TFT) desarrolla un sistema de diagnóstico de vehículos basado en bus CAN. El sistema implementa los protocolos ISO-TP (ISO 15765-2) como capa de transporte, OBD-II (SAE J1979) y UDS (ISO 14229-1) como capas de aplicación, ejecutándose sobre una Raspberry Pi 4 mediante scripts Python. Se implementan 25 PIDs del modo 0x01 (datos en tiempo real), la lectura del VIN del modo 0x09 (identificación del vehículo) mediante secuencia ISO-TP multi-frame, la gestión de DTCs en los modos 0x03 (lectura) y 0x04 (borrado), y dos servicios UDS: 0x10 (control de sesión) y 0x22 (lectura por identificador). La sesión de diagnóstico se expone de forma inalámbrica a través de un servidor BLE con perfil GATT, accesible desde una aplicación móvil nativa compatible con Android e iOS que ofrece monitorización en tiempo real, escaneo de DTCs y visualización de logs, con persistencia de las sesiones en una base de datos SQLite. El sistema se validó primero sobre un banco de pruebas basado en un simulador de ECU implementado en un Arduino MKR WAN 1310 y, posteriormente, sobre dos vehículos reales de fabricantes distintos: un Škoda Yeti de 2015 (grupo VAG) y un Opel Astra K (grupo PSA), confirmando la interoperabilidad de la capa OBD-II estandarizada entre marcas. En la monitorización continua el sistema sostuvo una cadencia de muestreo estable de entre 500 y 700 ms según el número de PIDs por ciclo, sin pérdida de muestras, y reconstruyó el VIN real del vehículo con un 100\,\% de fiabilidad. El diseño toma como referencia la arquitectura de comunicaciones del grupo VAG (Volkswagen Audi Group).}}

{\huge{\textbf{Abstract}}}

    {\large{This Final Degree Project (TFT) develops a vehicle diagnostics system based on the CAN bus. The system implements ISO-TP (ISO 15765-2) as the transport layer, and OBD-II (SAE J1979) and UDS (ISO 14229-1) as the application layers, running on a Raspberry Pi 4 via Python scripts. It implements 25 mode 0x01 PIDs (live data), VIN retrieval via mode 0x09 (vehicle ID) through an ISO-TP multi-frame sequence, DTC management in modes 0x03 (read) and 0x04 (clear), and two UDS services: 0x10 (session control) and 0x22 (read data by identifier). The diagnostic session is exposed wirelessly through a BLE server with a GATT profile, accessible from a native mobile application compatible with Android and iOS, offering real-time monitoring, DTC scanning, and log visualization, with session persistence in a SQLite database. The system was first validated on a test bench based on an ECU simulator implemented on an Arduino MKR WAN 1310 and later on two real vehicles from different manufacturers: a 2015 Škoda Yeti (VAG group) and an Opel Astra K (PSA group), confirming the interoperability of the standardized OBD-II layer across brands. During continuous monitoring the system sustained a stable sampling cadence of 500 to 700 ms depending on the number of PIDs per cycle, with no sample loss, and reconstructed the vehicle's real VIN with 100\,\% reliability. The design uses the communication architecture of Volkswagen Audi Group (VAG) vehicles as reference.}
